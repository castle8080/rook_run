# AI Guidance Index

This folder holds concise, stable guidance for AI coding work in this repository.

## Start Here

1. Read `project-map.md` for project responsibilities and where to make changes.
2. Read `coding-conventions.md` for code shape and boundary conventions.
3. Read the topic doc for the area you are changing.
4. Follow `feature-workflow.md` for planning, notes, testing, and review.

## Documents

- `project-map.md`: one-page map of projects, runtime flow, and technologies
- `coding-conventions.md`: C# and architecture conventions
- `exceptions-and-error-handling.md`: exception taxonomy and handling guidance
- `objectstore-paths-and-serialization.md`: canonical path, serialization, and indexing guidance
- `jobs-reliability-and-concurrency.md`: retry, throttling, idempotency, and race mitigation
- `feature-workflow.md`: implementation workflow and task notes pattern

## Maintenance

- Keep each file short and practical.
- Prefer links over long repeated explanations.
- Update docs in the same PR that changes behavior or conventions.
